# notes/ — 記事執筆の1次情報置き場

このディレクトリは「**ユーザーが書いた一次情報**」を保持するための領域。
Astro のビルド対象 (`src/content/posts/`) からは除外されているため、ここに何を置いても本番サイトには出ない。

## 構成

```
notes/
└── <article-slug>/
    ├── notes.md    # ユーザーの所感・思いついたこと (自由形式)
    └── images.md   # 画像インベントリ + 各画像の説明 (ユーザー記入)
```

## 役割分担

| ファイル | 編集者 | 内容 |
|---|---|---|
| `notes/<slug>/notes.md` | **ユーザー専用** | 構成・粒度問わず、思いついたまま記入 |
| `notes/<slug>/images.md` | **ユーザー専用** | 画像の中身を1〜2行で説明 (AIは画像を直接見ない) |
| `src/content/posts/<slug>.mdx` | **AI が編集** | `notes/<slug>/*` を元に推敲・構成した公開記事 |

## 反復フロー

1. ユーザー: `notes/<slug>/notes.md` を更新 (追記/修正/書き直し)
2. ユーザー: `notes/<slug>/images.md` の説明欄を埋める/差替指示を出す
3. AI: 上記2ファイル + `src/assets/posts/<slug>/` を見て mdx を書き直す
4. ユーザー: `npm run dev` のプレビューで確認 → 1 に戻る
5. 完成: `draft: false` + `noindex: false` に切替

## ルール

- AI は `notes/` 配下のファイルを **編集しない** (読むだけ)。例外: ユーザーが明示的に依頼した場合のみ
- 画像ファイル (`.jpg` / `.mp4` 等) は AI が直接 Read しない (トークン消費回避)。`images.md` の説明文を根拠にする
- 1次情報 / 2次情報の区別を保つことで、後から「自分の元の言葉」と「AIが整えた表現」を比較できる
